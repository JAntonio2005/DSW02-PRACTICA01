# Data Model: CRUD de Empleados

## Entity: Empleado

- Description: Registro principal de empleado para operaciones CRUD.
- Primary Key: `clave` (string, unique, required, generated, format: `EMP-` + auto-numeric)

## Fields

- `clave`: string, required, unique, generated by system, format `EMP-[0-9]+`
- `nombre`: string, required, maxLength=100
- `direccion`: string, required, maxLength=100
- `telefono`: string, required, maxLength=100

## Validation Rules

- Ningún campo puede venir vacío o solo con espacios.
- `nombre`, `direccion` y `telefono` tienen longitud máxima de 100 caracteres.
- `clave` se genera automáticamente con prefijo `EMP-` y consecutivo autonumérico.
- `clave` debe ser única; el consecutivo no puede reutilizarse.

## Relationships

- No hay relaciones obligatorias con otras entidades en este alcance.

## State Transitions

- No aplica estado de ciclo de vida adicional; entidad gestionada con creación, actualización y eliminación física.

## Persistence Notes

- Tabla objetivo: `empleados`.
- Índice/constraint único: `uk_empleados_clave` sobre `clave`.
- Secuencia para consecutivo: `empleados_seq`.
- Migración inicial gestionada con Flyway (`V1__create_empleados.sql`).
